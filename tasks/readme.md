# tasks

This directory contains implementation notes and historical task plans for
go-common changes.

## Files

- `enhancements.md`: review and validation workflow notes for repository-wide
  hardening work.
- `orm.md`: design and implementation notes for the `orm` module.

## Use For

- Understanding why a component was introduced or changed.
- Checking historical validation expectations before modifying the same area.

## Do Not Use For

- Public package API documentation.
- Runtime configuration.

## Gotchas

- Task notes can become stale. Prefer source code, tests, and package readmes
  for current behavior.
